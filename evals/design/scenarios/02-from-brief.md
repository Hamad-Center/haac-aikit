# Scenario 02 — Synthesize a design system from a verbal brief

## Task

The project is in early-stage greenfield development. The founder has provided
a verbal brief about the visual direction. Build a `DESIGN.md` file that
captures the design system implied by the brief, in marker-bounded sections so
every AI tool reading the repo can produce visually-consistent work.

## The brief

> "Quiet, generous whitespace, the kind of feel a museum guide pamphlet has.
> Off-white background — something between ivory and oat — and a deep ink for
> text, not pure black. The accent color should be a muted terracotta clay,
> warm but restrained. **Absolutely no gray** — I never want to see a gray
> rule, a gray text color, a gray card background. Use the clay or the ink at
> lower opacity if I need a secondary tone. Serif headlines, sans-serif body,
> mono for code. **No drop shadows under any circumstance** — hairline borders
> only. Generous spacing rhythm, around 64px between major sections. Single
> reading column, 720px max-width."

## Output instructions

Produce the **full content of `DESIGN.md`** and nothing else. No prefatory
commentary, no markdown code-fence around your output, no closing remarks.
The first character of your reply is the first character of the file.

The file should be readable by any AI tool that loads it as context, so favor
explicit, descriptive language. The project root will commit this file alongside
`AGENTS.md`.
