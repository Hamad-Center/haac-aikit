## Failure mode analysis

The freestyle baseline failed on the **structural** criteria (1, 2, 4) in
every scenario. These are exactly the gaps `/design` exists to close — the
marker engine's section IDs and the voice rule against design-token aliases
as the source of truth.

Freestyle passed on the **descriptive** criteria (3, 5, 6, 7) without
prompting. Hex codes in code ticks, font stacks, and numerical layout specs
appear to be natural defaults when an LLM writes a design system file. So
do not need to be enforced — what needs enforcement is the structural
contract.

The one with-skill failure (scenario 03, marker doubling) suggests the
SKILL.md should add a sentence specifically warning against wrapping the
file preamble inside the first section's markers. Worth a follow-up edit.

## Reproducibility

Run with: `Agent` tool from Claude Code, dispatching 6 parallel `general-purpose`
subagents each reading their scenario file (and `SKILL.md` for the with-skill
cells). Outputs captured in `outputs/`. Re-run any time by repeating the
dispatch — the scenario files are deterministic.

## Notes

- This run differs from the v0.13.0 result methodology in ways we can no
  longer audit (the original eval harness was lost). Numbers are not
  directly comparable; only the rough scale (+25 vs +29) is.
- 3 scenarios instead of 4 — the refine subcommand scenario was omitted
  because both conditions scored 100% in v0.13.0 (the marker engine works
  for both once shown the existing markers, so refine isn't where /design
  earns its keep).
- Single iteration per cell. A more rigorous re-run would do 5–10
  iterations per cell and report variance. The point of this pass was
  directional confirmation, not statistical rigor.
