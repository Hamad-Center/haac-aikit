v0.11.0 · npm

Two skills, not twenty templates

A small kit for keeping a project documented as it changes, and for writing one clean tradeoff page when there's a decision to make. That's it. We tried the 20-template version. It was too much.

$ npx haac-aikit
~157KB bundle cross-tool (7 agents) v0.11 breaking — see migration

Why we trimmed it down

The previous release shipped 20 forked HTML templates — PR reviews, slide decks, design systems, prototype animations, feature-flag editors, prompt tuners. A beautiful catalog. After living with it in real projects, the honest verdict was: it was too much overengineering for the everyday cycle.

Developers don't browse a 20-template menu when they're mid-task. A 205-line always-on skill is paid context on every single agent invocation, even when nobody's generating HTML. Most of the templates were never reached for in real life — they were a museum, not a workshop.

When we listed what people actually used HTML for, two patterns kept surfacing: living project documentation (for handover, onboarding, knowing what changed) and tradeoff pages (when there's a real decision to make). Everything else was a one-off that markdown could handle.

So /html got split into two focused skills, both shipping a single starter template each. This version covers 100% of the planning and project-docs work we kept reaching for — without the surface area we never used.

Want the old 20 templates? They're not gone, just not the default. Run git checkout v0.10.0 -- catalog/templates/html-artifacts/ in a clone of the repo and you have them back.

What lands

/docs — living project documentation

Always-on, ~80-line skill. Maintains an HTML doc tree at docs/: one rolled-up docs/index.html plus per-area files (architecture.html, auth.html, whatever you need). Updates are section-bounded — the agent reads and edits one named block at a time, not the whole file. Cheap, surgical, idempotent.

/decide — one rich tradeoff page per call

Opt-in (tier2), ~50-line skill. When you're about to make a non-trivial choice, generates a single rich HTML page comparing options: pros and cons as colored dots, side-by-side table, technical bit explained in plain language, recommendation marked with a star. Saves to docs/decisions/YYYY-MM-DD-<slug>.html as a permanent decision log.

The marker engine that makes it cheap

Documents are wrapped in <!-- BEGIN:haac-aikit:section:<id> --> blocks. Round-trip tested, idempotent, dialect-aware. Anything you write outside the markers — user-authored prose, hand-edited content — is preserved verbatim on every sync. This is what makes living docs viable without re-reading every file.

Using it

Two slash commands. The agent proposes before writing — no silent edits.

you: /docs the new auth refresh flow
claude:
→reads only the flow section in docs/auth.html
→"I'll update the auth flow section to reflect the refresh tokens — okay?"
→on yes, writes through the marker engine (everything outside markers preserved)
→updates the auth summary in docs/index.html too
you: /decide auth: session cookies vs JWT vs Clerk
claude:
→confirms options + recommendation with you in chat
→reads the tradeoff template, fills with real pros/cons
→writes docs/decisions/2026-05-13-auth-strategy.html

Both pages work on a phone, work with a screen reader, and can be opened in any browser — no build step, no CDNs.

Drop it into a project today

$ npx haac-aikit

30-second wizard. Writes .aikitrc.json you can commit. The full kit also drops in cross-tool rules, safety hooks, agents, and CI templates — but /docs and /decide are the two skills you'll touch every week.

Migrating from v0.10? Run aikit sync to pull the new templates, then delete the orphaned .aikit/templates/html-artifacts/ and docs/aikit-html-design-system.html if you have no local edits worth keeping.
