v0.10.0 · npm

One command for Claude Code's HTML output

Drop the html-artifacts skill and 20 production-ready templates into any existing repo. 2 seconds. No AGENTS.md, no MCP, no hooks — just HTML output.

$ npx haac-aikit init html
~250KB total no breaking changes drops into any repo

Why

After Thariq's piece on the unreasonable effectiveness of HTML for Claude Code, I tried setting it up in my own projects. It took longer than it should — picking templates, writing the skill, wiring the slash command.

So I packaged it into one npm install. Drops into any existing repo, old or new. The install is additive — it doesn't touch anything outside the html-artifacts paths, and the artifact output directory is gitignored automatically.

What lands

20 production-ready templates

Implementation plans, code reviews, status reports, design systems, prototypes, diagrams, decks, research explainers, editor UIs. Forked from @trq212's collection with permission.

A skill that uses them correctly

Teaches Claude Code to copy + edit the template instead of writing HTML from scratch (no design drift), put jargon in tag chips not in the prose, lead every plan with a decision callout at the top, and bake in accessibility for screen readers and mobile.

An auto-generated gallery

Every artifact you produce gets indexed at .aikit/artifacts/index.html, grouped by pattern. Your project accumulates a navigable archive of plans, reports, and reviews over time.

Using it

Once installed, the /html slash command does the rest:

you: /html plan the auth refresh
claude:
→picks the implementation-plan template
→confirms the artifact name with you
→if your inputs have mixed PROMOTE/HOLD/DEFER, asks which scope to render
→fills with real content (plain language in prose, jargon in chips)
→saves to .aikit/artifacts/NN-<topic>.html, regens the gallery

About 15 seconds end to end. The artifact opens with the decision callout at the top, scannable milestones, works on a phone, works with a screen reader.

Drop it into a project today

$ npx haac-aikit init html

If you don't have Claude Code yet, the templates still install — you can use them with any agent that can read a manifest and copy a file.
