The four HTML skills

Each one takes a single prompt and writes one self-contained HTML file you can open in a browser and commit to git. No build step, no CDN, no dependencies.

Safety + telemetry hooks

Hooks fire before a tool call. Safety hooks block dangerous operations; telemetry hooks observe rule adherence without blocking.

block-dangerous-bash.sh

Stops rm -rf /, chmod -R 777, and other shell footguns before they run.

block-force-push-main.sh

Refuses git push --force against main / master / develop.

block-secrets-in-commits.sh

Catches .env*, id_rsa*, AWS keys, and other secret patterns before they hit git.

file-guard.sh

Blocks reads of .ssh/, .aws/, secrets/, and *.pem files.

log-rule-event.sh

Records every time a rule loads or is cited. Powers the observability loop.

check-pattern-violations.sh

After writes, greps each file for rule patterns and flags violations as JSONL events.

judge-rule-compliance.sh

Optional LLM-judge step that scores whether the AI's output followed the cited rules. Off by default.

One file, seven AI tools see it

Write your rules once in AGENTS.md. The kit translates them into each tool's native format on every aikit sync.

one AGENTS.md → seven tool-native files, regenerated on every sync

Each translation is a dialect: Cursor MDC frontmatter, Claude emphasis tokens (IMPORTANT:, YOU MUST), Aider's imperative phrasing, Copilot's bullet style. Rules can carry paths=glob metadata to load only when matching files are touched.

And it goes deeper than rules — skills, agents, hooks, and MCP servers fan out per tool, each in the target tool's native format. The matrix below shows what gets installed when you select each tool:

Pick Cursor, get .cursor/rules/*.mdc for every skill, .cursor/hooks.json for safety hooks, and .cursor/mcp.json for MCP. Pick Codex, get .codex/agents/*.toml and .codex/config.toml [mcp_servers]. Each tool sees only its own files — no cross-tool ballast.

The observability loop

Rules without telemetry rot. Hooks log which rules load, which get cited, and which patterns get violated in just-written files. aikit doctor --rules shows the per-rule buckets; aikit report turns it into markdown or JSON.

declare → fire → measure → prune

Rules are bucketed as cited (the AI referenced it), loaded (in-context but not used), violated (pattern matched in a write), or dead (never observed). Dead rules cost tokens without payoff — the report flags them for removal.

Built on a real insight

The four HTML skills exist because Thariq Shihipar wrote "The Unreasonable Effectiveness of HTML for Claude Code." His point — that HTML output gives you scannable, interactive pages instead of chat transcripts — is what makes living docs, decision logs, design directions, and roadmaps actually worth shipping. We took the insight and built the four workflows that compound the longest, plus the cross-tool plumbing to keep them honest. — credit to @trq212 for the article and the design tokens we reuse.
