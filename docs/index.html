The two skills

/docs — living documentation

Maintains an HTML doc tree at docs/ — architecture, features, gotchas. The agent reads only the section it needs, proposes the edit, writes after you confirm. Everything outside the markers stays verbatim.

/decide — tradeoff pages

One rich HTML page per architectural choice. Options as cards, pros/cons as colored dots, recommendation marked with a star, technical bit rewritten for non-engineers. Each call appends to a permanent decision log.

The marker engine that makes it cheap

Sections wrapped in <!-- BEGIN:haac-aikit:section:<id> --> blocks. The agent loads one block per edit, not the whole file. Round-trip tested, idempotent, dialect-aware.

In practice

you: /docs the new auth refresh flow
claude:
→reads the flow section in docs/auth.html
→"I'll update the auth flow to reflect refresh tokens — okay?"
→on yes, writes the section + updates index.html summary
you: /decide session cookies vs JWT vs Clerk
claude:
→confirms options + your recommendation in chat
→writes docs/decisions/2026-05-13-auth-strategy.html

Built on a real insight

These skills exist because Thariq Shihipar wrote "The Unreasonable Effectiveness of HTML for Claude Code." His point — that HTML output gives you scannable, interactive pages that feel like tools instead of chat transcripts — is what makes living docs and decision logs actually worth shipping. We took the insight and built the workflows that compound the longest. — credit to @trq212 for the article and the design tokens we reuse.
